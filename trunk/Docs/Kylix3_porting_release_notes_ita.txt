InstantObjects: Delphi -> Kylix 3 porting release notes
-------------------------------------------------------

Aggiornato al 21/5/2004:

E' stato fatto il porting del Primer! Nella cartella PrimerCross c'è la versione cross-platform.
Si è usata la stessa tecnica per il porting delle form nell'IDE: il dfm è lo stesso, e le proprietà incompatibili vengono gestite a run-time.
Solo la form HelpView è stata duplicata in HelpViewK3: invece di un componente TWebBrowser viene usato un TTextBrowser.

LA NUOVA VERSIONE USA I TFRAME
Oltre a questo il lavoro più grosso non era legato a InstantObjects: in primer vengono usate delle TForm messe a run-time alClient su un pannello della Main.
Questa cosa non funziona assolutamente in Kylix! Si è dovuta cambiare logica usando i TFrame e gestendo la Caption e la FormCreate a mano.

PROBLEMA CON UN FILE DI RISORSE DI IMMAGINI
Si è riscontrato un problema con le immagini: così come per le form nell'IDE tutte le imagelist vengono riempite a run-time da un file di risorse.
Stranamente sotto Kylix il file di risorse non viene linkato generando un errore incomprensibile.
Sotto Kylix ho dovuto caricare le immagini da file esterni.
N.B. per poter compilare PrimerK3.dpr occorre scaricare e installare TChart components open edition (su Torry).
Se non si vuole installarlo basta togliere la unit PerformaceView dal progetto e non crearla nel main.

FILE DI CONFIGURAZIONE CONNECTION MANAGER IN XML
In parallelo è stata aggiunta la possibilità di salvare i file di connessione del connectionmanager in formato XML: basta specificare sfXML in FileFormat.
A run-time questo va fatto prima di assegnare FileName al connectionmanager (in main.pas c'è l'esempio remmato).

Descrizione modifiche a InstantObjects per porting a Kilix 3.

A) USO DI UN UNICO FILE DFM E INCOMPATIBILITA' DELPHI/KYLIX
1) Su alcune Form integrate nell'IDE si è reso necessario fare alcune piccole modifiche per evitare che componenti all'interno della form mostrassero nel dfm proprietà incompatibili.
Per le proprietà incompatibili si è proceduto togliendole dal dfm (impostando quindi il default) e mettendo l'istruzione di inizializzazione nell'evento FormCreate (opportunamente sotto IFDEF).

2) Un altro problema si è rivelato con le immagini contenute nelle imagelist: si sono dovute "esportare" in file di risorse e caricate a run-time: solo così sia Delphi che Kylix inizializzano correttamente il colore di sfondo.

3) I font sono stati impostati tutti a "parentfont true" e sono stati impostati nel FormCreate.

4) Per i componenti che sfruttavano l'anchor (e quindi creavano la proprietà DesignSize nel dfm) si è proceduto con tecniche alternative all'anchor: 
- per i pulsanti in basso a destra sono stati collocati su un pannello alRight (ButtonsPanel) dentro un pannello alBottom (BottomPanel).
- per gli edit con anchor a destra, si è scritto l'evento onresize.
N.B. questo sistema ha consentito di mantenere un unico file (dfm) compatibile tra Delphi e Kylix. 

5) Nei files .pas sono stati introdotti IFDEF MSWINDOWS e IFDEF LINUX nelle porzioni di uses.

ATTENZIONE: per mantenere la compatibilità cross-platform, le modifiche alle form integrate nell'IDE vanno eseguite con queste regole:
- Usare sempre Delphi (Kylix introduce delle proprietà che vengono salvate nel file dfm e che non sono compatibili con Delphi).
- Modificare i font e i color solo nel .pas (deve essere tutto ParentFont e ParentColor a true)
- Modificare borderstyle solo nel .pas
- Usare componenti esistenti anche per Kylix
- Le ListView hanno diverse incompatibilità: fare particolare attenzione.

B) ELENCO DEI FILES CREATI
- design/iodesK3.dpk: package InstantObjects per Kylix3
- in source/images ci sono le immagini usate nelle form
- in design/iodesimages.res ci sono le stesse immagini
- in core/connectionmanagerimages.res ci sono le immagini per il connection manager
- core/InstantImageUtils.pas: routine di caricamento immagini a run-time dalle risorse
