Frequently Asked Questions

Here are answers to some of the most common questions asked about InstantObjects™.

Q:
Does InstantObjects support legacy databases?
A: Yes, it is possible to adapt existing databases to InstantObjects.

Q:
How does InstantObjects store objects in a relational database?
A: InstantObjects uses a mapping scheme to store objects in the database. This mapping scheme is defined by specifying storage names for the persistent classes and their attributes.
When persisting objects to a relational database, the storage name for a class specifies the table and the storage name for an attribute specifies a field in the table. Several classes may share the same table by defining the same storage name.

Q:
How does InstantObjects handle object relations?
A: InstantObjects supports relationships in two flavours; part relationships (Composition) and reference relationships (Association). When mapped to relational databases, parts are stored in BLOB fields, meaning they are only available through the owning object. References, on the other hand, refer to instances of objects outside the referring object. If you want attributes of objects included as parts to be available at query level, you can either surface the attributes by replicating them in the owning object or use a reference instead.

Q:
Can InstantObjects be used in multi-user environments?
A: Yes, InstantObjects uses an optimistic locking protocol and supports transactions.

Q:
What about third-party databases?
A: Support for various databases has been one of the main design goals of InstantObjects. Currently , the following database access layers are supported:
  • Borland Database Engine
  • ActiveX Data Objects
  • InterBase Express
  • Advantage Database Server
  • ElevateSoft DBISAM
  • TurboPower FlashFiler

The persistence mechanism of InstantObjects actually supports any database that can be accessed through TDataSet descendants. Enabling InstantObjects for other databases is simply a matter of creating a specialized object broker for the specific database.

Q:
What about third-party data-aware controls and reporting tools?
A: InstantObjects supports any control and reporting tool capable of accessing data through TDataSet descendants.

Q:
Does InstantObjects support XML?
A: Yes, the InstantObjects streaming system is capable of reading and writing objects in the XML format.
